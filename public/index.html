YomiAPI - Documentación

La peor API sobre Hiragana & Katakana

¿De qué trata esta API?

A continuación explico de qué trata esta API y un poco cómo funciona el sistema de escritura japonés, aunque si querés podes pasar directamente a la parte de los endpoints

Breve Descripción

Esta API REST permite trabajar con Hiragana y Katakana, los dos alfabetos básicos del idioma japonés.
Su objetivo es ofrecer datos estructurados (caracteres, sonidos, ejemplos) para que desarrolladores, estudiantes o aplicaciones de aprendizaje puedan mostrar, buscar, practicar o dar sus primeros pasos con estas escrituras.

Contexto en pocas palabras

El japonés se escribe combinando tres sistemas:

  1. Hiragana (ひらがな): un alfabeto silábico utilizado para escribir palabras nativas, partículas gramaticales y terminaciones verbales.
  2. Katakana (カタカナ): otro alfabeto silábico, usado principalmente para nombres extranjeros, onomatopeyas y palabras prestadas de otros idiomas.
  3. Kanji (漢字): caracteres de origen chino que representan conceptos o palabras completas. (Kanji no está incluido en esta API debido a su absurda complejidad, pero si te interesa "Jisho" está trabajando en su propia API).

¿Qué ofrece la API?

  • Listado completo con todos los Hiraganas y Katakanas (sin contar los deprecados)
  • Mnemotecnias
  • (Para futuras versiones) Pronunciación
  • (Para futuras versiones) Orden de los trazos
  • (Para futuras versiones) Parseo de palabras completas
  • (Para futuras versiones) Vocabulario

Endpoints

Usuarios
GET /api/v1/users/
GET /api/v1/users/:id
POST /api/v1/users 
{
  name: String,
  email: String
  password: String
}
PUT /api/v1/users/:id Todos los valores especificados en el "POST" son editables
DELETE /api/v1/users/:id
Hiragana & Katakana
GET /api/v1/kanas
GET /api/v1/kanas/parse/:valor/to/:parseo Por el momento sólo se puede parsear 1 único valor, pero se estima que para futuras versiones se podrá parsear palabras completas
Mnemotecnias
GET /api/v1/mnemonics
GET /api/v1/mnemonics/:id
GET /api/v1/mnemonics/kana/:kana
POST /api/v1/mnemonics/ 
{
  kana: String,
  descr: String, // Descripción
  main_img: String, // Imagen principal
  kana_img: String, // Imagen con el kana que encastre con la principal
  tags: Array, // Lista de etiquetas
  lang: String
}
PUT /api/v1/mnemonics/:id Todos los campos del "POST" son editables
DELETE /api/v1/mnemonics/:id
